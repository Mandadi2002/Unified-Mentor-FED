🚀 How to Deploy Your Website for Free Using GitHub Pages

So, you’ve built a cool website using HTML, CSS, and maybe even some JavaScript.
What now? You probably want the world to see it, right?
Well, the good news is—you can host your website online for free using GitHub Pages.
No need to buy a domain or set up a complex server.
In this blog, I’ll walk you through the process step by step.

💡 What is GitHub Pages?

GitHub PagesGitHub Pages is a free hosting service provided by GitHub that lets you
publish static websites directly from your GitHub repository.
It’s perfect for portfolios, documentation, blogs, and simple web projects.
🧰 What You’ll Need:

  • A GitHub account (sign up at github.com if you don't have one).

  • A completed website (basic files like index.html, style.css, script.js, etc.).

  • Git installed on your computer (optional but helpful).

    • 🪜 Step-by-Step Deployment Guide:

    ✅ Step 1: Create a GitHub Repository

    1.Go to github.com and log in.
    2.Click the “+” icon in the top-right corner → New repository.
    3.Give your repo a name (e.g., my-portfolio).
    4.Choose Public.
    5.Check the box to initialize with a README (optional).
    6.Click Create repository.

    ✅ Step 2: Add Your Project Files

    Upload Files via GitHub UI
    1.Open your repo.
    2.Click “Add file” → “Upload files”.
    3.Drag and drop your project files (including index.html).
    4.Commit the changes.

    ✅ Step 3: Enable GitHub Pages

    1.In your repo, go to Settings (top right).
    2.Scroll down to Pages (left sidebar).
    3.Under "Source", select main branch and / (root) folder.
    4.Click Save.

    After a few seconds, you’ll see a green message like:
    Your site is published at https://your-username.github.io/my-portfolio/
    Boom 💥 — your site is live!

    📌 Bonus Tips:

  • >Make sure your main file is named index.html — GitHub Pages loads this by default.

  • Use relative paths for images and CSS/JS files (e.g., ./style.css).

  • To update your site, just push new changes to your GitHub repo.


    • 📦 Use Cases:

  • Portfolio Websites

  • Project Showcases

  • Documentation Pages

  • Personal Blogs (even with Jekyll)
